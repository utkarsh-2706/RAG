Vector Database Guide

Vector databases are purpose-built to store, index, and query high-dimensional embedding vectors. They power semantic search, recommendation systems, and RAG pipelines.

What Are Embeddings?

Embeddings are dense numerical representations of data. Text, images, and audio can all be converted into embedding vectors. The key property is that semantically similar items have vectors that are close together in the embedding space.

A 384-dimensional embedding vector looks like a list of 384 floating point numbers. The meaning is encoded in the geometric relationships between vectors, not in any single dimension.

How Vector Search Works

Given a query, a vector database finds the stored vectors that are most similar to the query vector. Similarity is typically measured using cosine similarity or dot product.

Approximate Nearest Neighbor Search

Exact nearest neighbor search over millions of vectors is too slow for real-time applications. Approximate Nearest Neighbor (ANN) algorithms trade a small amount of recall for massive speed gains.

HNSW (Hierarchical Navigable Small World) is the most popular ANN algorithm. It builds a multi-layer graph where each layer is a progressively sparser view of the data.

Filtering and Hybrid Search

Pure vector search retrieves by semantic similarity alone. In practice, you often need to combine vector search with structured filters — for example, retrieve documents similar to a query but only from a specific date range or category.

Hybrid search combines vector similarity with BM25 keyword scoring. This is powerful for queries that contain specific terms like product codes or proper nouns that embeddings may not capture well.

Popular Vector Databases

The ecosystem has grown rapidly. Here are the main options and their trade-offs.

FAISS

Facebook AI Similarity Search is a library, not a database. It runs in-process and is ideal for offline batch processing or when you want full control over the index. No server, no network overhead.

Chroma

Chroma is an open-source, developer-friendly vector database. It runs locally with a simple Python API and is popular for prototyping. It persists to disk and supports metadata filtering.

Pinecone

Pinecone is a fully managed cloud vector database. It scales to billions of vectors automatically and offers low-latency queries globally. Best for production systems that need to scale without managing infrastructure.

Choosing the Right Database

For local development and prototyping, use Chroma or FAISS. For production at scale, evaluate Pinecone or Weaviate based on your needs for hybrid search, filtering complexity, and operational burden tolerance.
